# 实施方案模板（习惯一）

> 用于「先出方案再动手」习惯。Agent 在动工前必须输出符合本模板的方案给用户 Review。

---

## 1. 需求理解

> 用一句话复述用户的核心诉求，避免歧义。

## 2. 涉及的文件与改动点

| # | 文件路径 | 改动类型 | 说明 |
| - | --- | --- | --- |
| 1 | `src/foo.ts` | 新增 / 修改 / 删除 | 一句话说明 |

## 3. 实施步骤

1. 第一步：……
2. 第二步：……
3. 第三步：……

## 4. 潜在风险 / 影响面

- 风险 1：……（缓解方案：……）
- 风险 2：……（缓解方案：……）

## 5. 验证方式

- [ ] 单元测试：……
- [ ] 手工验证步骤：……
- [ ] 其他：……

---

> ⚠️ **请 Review 上述方案**，确认无误后回复「通过 / OK / 可以」我再开始执行。如有调整请直接指出，我会修改方案后再次提交 Review。
